# 💰 Cost Analysis & ROI

Our 3-layer solution is designed to be highly cost-effective, leveraging existing software infrastructure wherever possible before deploying hardware.

## Cost Breakdown (for 3,00,000+ Merchants)

| Layer       | Solution                            | Capital Expenditure (CAPEX) | Operating Expenditure (OPEX) | Details                                                                                                       |
| :---------- | :---------------------------------- | :-------------------------- | :--------------------------- | :------------------------------------------------------------------------------------------------------------ |
| **Layer 1** | Rider Location & Accuracy Dashboard | **₹0**                      | ₹0                           | 100% Software update. Leverages existing rider GPS telemetry.                                                 |
| **Layer 2** | Barcode Verification (Bottom 5%)    | **₹1.5 Crore (Max)**        | Minimal                      | Thermal printers (₹3000) for ~5000 large bottom-tier offenders. App-based QR for medium/small merchants (₹0). |
| **Layer 3** | Kitchen Rush Slider & POS Sync      | **₹0**                      | Variable API Costs           | Slider is a UI element (₹0). POS integration relies on existing 3rd party aggregators (UrbanPiper, Petpooja). |
| **TOTAL**   |                                     | **< ₹2 Crore**              |                              | A fractional cost compared to the current losses from idle time.                                              |

## The ROI: Cost vs. Rider Idle Time

Currently, an average rider waits 12.3 minutes per order. If we reduce this by just 4 minutes (as simulated) across 2 Million orders a day, we save **133,000 Rider Hours EVERY SINGLE DAY**. The millions of rupees saved in rider payouts (wait-time compensation) and increased rider efficiency will pay for the Layer 2 hardware within the first two weeks of rollout.
